# visPedigree: 算法架构与功能创新点凝练

> 本文档用于为后续学术论文或技术报告提供核心写作素材

## 1. 算法创新与底层计算架构突破

### 1.1 高效的拓扑整数图排序与 C++ 计算引擎

传统的系谱遍历算法（如基于字符型 ID 的递归查找）在处理超过数十万条记录（如 >200,000 条水产或家畜系谱）时，常面临指数级的时间复杂度爆炸和内存溢出。
**创新点**：`visPedigree` 在数据预处理阶段（`tidyped()`）彻底抛弃了低效的标签递归。引入了严格的时间拓扑排序，并为每个个体生成了唯一的递增整数索引（`IndNum`, `SireNum`, `DamNum`），使后续所有的世代流转计算可以直接映射为 C++ 层的连续内存数组操作（$O(N)$ 复杂度），从根本上避免了超深系谱下的系统假死挂起。

### 1.2 "幽灵亲本"（Phantom Parents）概率质量守恒修正

在实际育种系谱中，由于记录丢失或野外引入，“单亲本缺失”（如知父不知母）极易导致基因溯源时丢失一半的概率质量（Gene Leakage）。
**创新点**：我们在 `pedcontrib()` 底层算法中首创引入了“幽灵亲本”插补校正机制。当检测到半始祖（Half-founders）时，程序在 C++ 内存态临时构建对应的虚拟配合亲本，从而保证基因贡献概率的绝对守恒（所有节点流向下代的概率总和强逼近至 1.000）。极大地提高了测算非完整系谱时 $f_e$（有效始祖数）的稳健性。

### 1.3 边际贡献动态折减（Boichard's Iterative Algorithm 深度实现）

计算 $f_a$（有效祖先数）的核心难点在于防止漏斗谱系中“血统叠加”导致的重复计算。
**验证与创新**：程序严格实现了 Boichard (1997) 的迭代算法。通过动态扣除已被先前超级祖先解释的血统路径，能够极其敏锐和准确地识别出瓶颈个体（Bottlenecks）。在模拟人工极端漏斗模型中，测试输出 $f_a = 2.6667$ 与纯理论数学计算完美咬合，证明算法逻辑达到了“零计算重叠”的严谨度。

## 2. 核心功能集成与多维度分析体系

### 2.1 高度集成的多维遗传监控面板 (`pediv`)

计算一个育种群体的遗传状况通常需要在不同函数间反复传递变量，增加了二次错误的几率。
**创新点**：我们设计了 `pediv()`（Pedigree Diversity）作为单一包装器（Wrapper）入口。只需输入 `tidyped` 对象并指定研究人群（`reference`），即可**一键异步执行**并返回多维核心指标整合报告：

- 等位基因流失探测（$f_e$ 和 $f_a$ 的比对）；
- 世代流转指标；
- 多种假设前提下的 $N_e$ (Effective Population Size)。

### 2.2 三径并行的有效群体含量测算 (Multi-path $N_e$)

针对有特定选择压力的群体盲目套用公式可能导致的误差，我们实施了更为精细的 $N_e$ 评估框架（`pedne()`），支持以下三种并行方法供研究者对齐理论：

1. **基于协亲缘率（Coancestry, $\Delta c$）**：作为核心默认方法，深刻捕捉了由于选择偏差带来的潜在衰退，被视为受选育种群的“黄金标准”。
2. **基于近交率（Inbreeding, $\Delta F$）**：追踪表型观测到的实际近交情况。
3. **基于人口学（Demographic）**：以 $N_m$ 和 $N_f$ 提供基础管理视角的静态参照。
   *注：对于超大矩阵协亲缘计算，更引入了 OpenMP 多线程支撑，极大地扩展了可操作边界。*

## 3. 术语标准化与设计哲学 (API Standardization)

**API 严谨度优化**：将原有通俗但模糊的 `cand` (Candidate) 参数，全线全面重构为国际群体遗传学通用的术语 `reference` (Reference Population)。这一转变不仅是 API 面貌的改进，更是本软件包在统计学定位上的深思熟虑——即我们所评估的永远是由某一批参考子代逆向推演回溯其群体演解过程的系统全貌。

## 总结寄语 (Summary for Manuscript)

通过提供基于拓扑序内存优化的 C++ 计算层，并完美填补单亲代概率流失与重叠血统路径计算中的算法缺陷，`visPedigree` 为全基因组时代的常规繁育群体提供了一款兼具**严密数理逻辑、极低内存负载及直观报告生成**的现代型系谱组学监控工具。
